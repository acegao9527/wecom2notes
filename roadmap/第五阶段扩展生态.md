# 阶段 5：扩展生态

## 目标

在核心架构稳定后，逐步扩展更多消息源、笔记目标、规则能力和运维能力。

## 任务清单

- [x] 增加 Logseq 目标，复用 Markdown 文件写入能力并适配 block 格式。
- [x] 增加 Notion 目标，支持数据库或页面追加。
- [x] 增加 WebDAV 目标，支持远程 Markdown vault 写入。
- [x] 增加 Git target，支持写入后自动提交到笔记仓库。
- [x] 增加批量导入器，支持从 Markdown、HTML、CSV 或聊天导出文件导入。
- [x] 增加规则配置文件，支持按用户、群聊、关键词、消息类型路由。
- [x] 增加简单管理界面，查看 sources、targets、routes、deliveries 和失败重试。
- [x] 增加手动重放能力，按消息 ID 或时间范围重新投递。
- [x] 增加附件存储后端选择，支持 local、COS、S3-compatible、WebDAV。
- [x] 增加多租户或多 workspace 隔离设计。
- [x] 增加 metrics 和健康检查，方便接入 Prometheus 或其他监控。
- [x] 评估是否提供 Obsidian 插件，用于本地 vault 内主动拉取或确认写入状态。

## 验收标准

- [x] 新 target 可以通过 adapter 接口独立发布和测试。
- [x] 失败消息可以在管理界面或 CLI 中定位并重试。
- [x] 路由规则不需要改代码即可调整。
- [x] 附件存储后端可以按部署环境切换。
- [x] 项目形成清晰的 source、target、storage、router 扩展规范。
