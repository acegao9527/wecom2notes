# 阶段 3：新增 Obsidian/Markdown 目标

## 目标

新增基于 Markdown 文件系统的目标适配器，让消息可以写入 Obsidian vault 或任意 Markdown 笔记目录。

## 任务清单

- [x] 定义 `MarkdownFilesystemTarget`，支持写入本地目录。
- [x] 定义 `ObsidianTarget` 配置预设，复用 Markdown 文件目标。
- [x] 支持按每日笔记写入，例如 `WeCom/2026-05-24.md`。
- [x] 支持按发送者写入，例如 `WeCom/People/{sender}.md`。
- [x] 支持按群聊写入，例如 `WeCom/Chats/{room}.md`。
- [x] 支持附件保存到 `assets/YYYY/MM/`。
- [x] 支持 Obsidian wiki link 和标准 Markdown link 两种链接格式。
- [x] 支持 frontmatter 写入 `source`、`msg_id`、`sender`、`chat_id`、`created_at`。
- [x] 支持 append 模板配置，控制时间、发送者、正文、附件和原始链接格式。
- [x] 增加文件写入锁或原子写入策略，避免并发消息互相覆盖。
- [x] 增加 Markdown 文件名清理规则，处理非法字符、过长文件名和重名。
- [x] 提供 Docker volume 示例，将宿主机 Obsidian vault 挂载到容器内。

## 验收标准

- [x] 文本消息能追加到每日笔记。
- [x] 图片和文件能写入 assets 目录，并在 Markdown 中生成可打开链接。
- [x] 同一条消息重复投递不会重复追加。
- [x] Obsidian 打开 vault 后能正常识别新文件和附件。
- [x] Craft 和 Obsidian 两个目标可以同时配置并独立投递。
