# 阶段 4：开源发布

## 目标

把项目整理成其他人可以理解、部署、贡献和扩展的开源服务。

## 任务清单

- [x] 确认项目名、包名、镜像名和文档名统一为 `wecom2notes`。
- [x] 补充 `LICENSE`，默认采用 MIT 或 Apache-2.0。
- [x] 清理 `.env.example`，确保没有真实服务标识或敏感配置。
- [x] 重写 README，包含项目定位、支持矩阵、快速开始、架构图和安全边界。
- [x] 新增 `docs/wecom.md`，说明企业微信会话存档配置。
- [x] 新增 `docs/craft.md`，说明 Craft 目标配置。
- [x] 新增 `docs/obsidian.md`，说明 Obsidian/Markdown 目标配置。
- [x] 新增 `docs/deployment.md`，说明 Docker Compose、本地开发和生产部署。
- [x] 新增 `docs/troubleshooting.md`，沉淀常见错误和排查步骤。
- [x] 新增 `CONTRIBUTING.md`，说明开发流程、测试要求和 adapter 贡献方式。
- [x] 新增 GitHub Actions，执行 lint、类型检查、单元测试和 Docker build。
- [x] 增加 release checklist，确保发布前完成安全检查和迁移说明。

## 验收标准

- [x] 新用户只看 README 和 `.env.example` 可以完成最小启动。
- [x] CI 在 Pull Request 上自动运行并能拦截基础错误。
- [x] 仓库中没有真实 token、secret、私钥、数据库和本地运行数据。
- [x] 文档清楚说明个人微信自动抓取不属于首期稳定能力。
- [x] GitHub 仓库名称、README 标题和 Docker 镜像名一致。
